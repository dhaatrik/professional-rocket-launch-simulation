# Contributing to DeltaV Lab

First off, thank you for considering contributing to **DeltaV Lab**! It's people like you that make professional open-source tools reality.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

1. Ensure the bug was not already reported by searching on GitHub under [Issues](https://github.com/dhaatrik/professional-rocket-launch-simulation/issues).
2. If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.

1. Check to ensure the enhancement hasn't been suggested already. 
2. Open a new Issue describing the enhancement. Explain why this enhancement would be useful to most users.

### Submitting a Pull Request
1. Fork the repository and create a new branch from `main`.
2. Make your proposed changes. Ensure any new logic is accompanied by tests.
3. Run `npm run lint` and `npm run format` locally before submitting.
4. Issue a pull request against the `main` branch.

## Code of Conduct

By participating in this project, you are expected to uphold a welcoming, respectful, and collaborative environment.

Thank you for contributing!
