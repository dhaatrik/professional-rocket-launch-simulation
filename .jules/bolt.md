## 2025-02-25 - Prevent Path2D Allocations in Render Loops
**Learning:** Instantiating `new Path2D()` on every draw call within high-frequency rendering loops (like telemetry graphs) causes significant GC overhead and degrades frame rates.
**Action:** Replace `new Path2D()` allocations with direct Canvas context mapping (`beginPath`, `moveTo`, `lineTo`, `stroke`). When drawing multiple distinct lines, separate the data iterations into distinct loops for each line rather than combining them, as the standard Canvas API only supports one active path at a time. This avoids thrashing the canvas state.
